description: 実装後の必須チェックループ（サブエージェントで指摘0件まで反復）
---

直前の実装に対して、**実装後チェックループ**（.claude/rules/build-commands.md）を実行してください。

手順:

1. チェック専用のサブエージェント **`reviewer`（Sonnet 5 固定）** を `Agent` で起動する（既存/作成済みファイルへの実行のため：CLAUDE.md 原則7）。以下の観点で直近の変更を確認させる:
   - 修正漏れ・要件（Notion / docs）との齟齬
   - 不具合（バグ、例外処理漏れ、型不整合など）
   - 冗長なコード（重複ロジック、不要な再レンダリング、デッドコード）
   - `docs/` 設計書との整合性
   - git のマージ／PR 先が `develop` になっているか（.claude/rules/git-workflow.md）
2. 指摘リスト（該当ファイル:行・種別・修正案）を受け取る
3. 指摘が **1件以上** → 修正し、手順1へ戻って再チェック
4. 指摘が **0件** → ループ終了として日本語で完了報告
