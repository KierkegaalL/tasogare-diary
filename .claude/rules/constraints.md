# constraints.md — 制約事項・非機能要件

## パフォーマンス

- 画面遷移・入力は 60fps を目標とし、体感の引っかかりを出さない。
- 「こころの灯」オーブのアニメーションは可能な限り `react-native-reanimated`（UIスレッド駆動）で実装し、JS スレッドをブロックしない。
- Claude API 応答は非同期で処理し、待機中はローディング／プレースホルダ表現を出す（フリーズさせない）。
- リスト表示（過去の日記等）は仮想化（`FlatList` 等）を用い、大量データでも軽量に保つ。

## オフライン対応

- 日記の下書き・入力は **オフラインでも継続可能** とする（ローカル保存 → オンライン復帰時に同期）。
- Firestore のオフライン永続化を活用する。
- Claude API 応答（ステップ3）はネットワーク必須。オフライン時はその旨を明示し、下書きは保存しておく。

## プライバシー・データ保持方針

- 日記は極めてセンシティブな個人情報。**Claude API キーはクライアントに埋め込まず、必ずサーバ側プロキシ経由で呼び出す**（実装は Cloudflare Workers。Firebase Blaze プラン回避のため Firebase Functions は不採用。[environments.md](environments.md)）。
- ユーザーデータは本人のみアクセス可能とする（Firestore セキュリティルールで uid スコープを強制）。
- 日記本文を分析・学習等の二次利用に回さない。Claude API へは応答生成に必要な最小限のみ送信する。
- データ保持: ユーザーは自分の日記・アカウントを削除でき、削除時は関連データ（日記・ペアリング情報）も確実に削除する。
- ログ・分析に日記本文や個人特定情報を残さない。

## その他

- アクセシビリティ: 文字サイズ変更・十分なコントラスト（たそがれ配色でも可読性を確保）に配慮。
- 対応言語: まず日本語。将来的な多言語化を阻害しない実装にする。
- セキュリティルール・Functions の権限は最小権限を原則とする。
